かんなびメモランダム

KANNABI MEMORANDUM

概要

『かんなびメモランダム』 は、
オリジナル企画の制作過程において行われた
ナラティブ設計・構成設計・レビュー工程を対象とした
制作実験の記録および方法論の公開リポジトリです。

本リポジトリで扱うのは、
物語そのものではなく、

どのように設計したか

どのような課題が発生したか

どの判断基準で修正したか

という 制作プロセスと意思決定の履歴です。

本リポジトリの目的

ナラティブ制作工程をブラックボックス化しない

個人制作における設計・レビュー手法を言語化する

第三者が検証・再現可能な形で制作過程を残す

これは完成形のノウハウ集ではなく、
**設計と思考の記録（メモランダム）**です。

本文（成果物）について

本企画で制作された 完成した物語本文（成果物） は、
GitHub 上では公開していません。

本文は ブログ媒体にて成果物として公開しています。

GitHub：制作プロセス・設計・判断の記録

ブログ：読まれるための成果物（本文）

という役割分担を採用しています。

👉 本文をどのように作ったかを知りたい方は、
このリポジトリを参照してください。

公開ポリシー（要約）

本リポジトリは、以下を原則とします。

成果物だけでなく、途中経過・失敗・廃案も含めて公開する

後付けの成功談ではなく、実際の判断履歴を残す

情報は「編集済み」と「一次資料」に分けて整理する

詳細は後述の構成をご覧ください。

リポジトリの構成と読み方
はじめての方へ（3ステップ）

現在の推奨構成を確認する
→ docs/00_current_state.md

全体像を把握する
→ docs/01_overview.md

関心のある工程を読む
→ Config / シーンカード / レビュー など

ディレクトリ構成
.
├─ docs/        # 編集済みドキュメント（読む）
├─ templates/   # 再現用テンプレート（使う）
├─ raw/         # 一次資料・実験ログ（検証する）
├─ archive/     # 廃案・非採用案（判断を学ぶ）
└─ diagrams/    # 説明用図版

各ディレクトリの役割
docs/（編集済みドキュメント）

第三者向けに整理された解説資料です。

なぜその手法を採用したか

どのような課題があったか

どう判断・修正したか

を、工程ごとに記録しています。

原則として docs だけを読めば全体像が理解できる構成を目指しています。

templates/（テンプレート）

実際に使用した、または使用可能なテンプレート群です。

Config 設計

シーンカード

テンション曲線

クリニカルレビューシート

ここからコピーして、自身の制作に利用できます。

raw/（一次資料・実験ログ）

制作中に実際に使用したデータを、
可能な限りそのまま収録しています。

Config の実例

シーンカード

レビューコメント

判断ログ

すべての資料には、
何のために使われ、何が分かったかという
最低限のメタ情報を付与しています。

archive/（廃案・非採用案）

採用に至らなかった設計案・手法をまとめています。

なぜ機能しなかったのか

なぜ採用しなかったのか

を記録することで、
設計判断の背景を残すことを目的としています。

このリポジトリの使い方

制作工程を設計する際の参考例として

ナラティブ制作のレビュー手法を検討するために

個人制作・小規模チーム制作の実験事例として

必要な部分だけを切り出して利用してください。

注意事項

本リポジトリは特定の制作成果を保証するものではありません

手法はプロジェクト規模・目的に応じて調整が必要です

内容は進行中のプロジェクトとして更新されます

最後に

本リポジトリは
「正解の提示」ではなく、
制作における思考と判断の過程を共有する試みです。

誰かが制作の途中で立ち止まったとき、
「他者はどう考え、どう判断したか」を
確認できる場所になることを意図しています。
