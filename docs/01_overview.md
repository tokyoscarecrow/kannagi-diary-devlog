# 制作プロセス全体像（Overview）

## このドキュメントの目的
本ドキュメントは、  
『かんなぎダイアリー』制作における  
**全体的な制作フローと各工程の役割関係**を俯瞰するための概要資料です。

個別の設計思想や運用結果については、  
各工程ごとのドキュメントで詳述します。

---

## 制作フローの全体像（抽象）
本プロジェクトでは、物語制作を以下の工程に分解しています。

1. **Config 設計**  
   世界観・制約・評価軸を定義する
2. **シーンカード設計**  
   各シーンの機能・感情・情報開示を設計する
3. **本文生成／執筆**  
   設計に基づいて物語本文を作成する
4. **レビュー／評価**  
   設定・感情・主題の整合性を検証する

これらは直線的ではなく、  
必要に応じて往復・修正されていきます。

---

## 各工程の役割

### Config 設計
- プロジェクト全体の前提条件を固定する
- 世界観・能力体系・倫理ルールを明文化する
- 生成・レビュー時のブレを抑制する

→ 詳細は `docs/10_config_design.md` を参照。

---

### シーンカード設計
- シーンを「出来事」ではなく「機能」として定義する
- 感情の遷移と情報開示を制御する
- 前後シーンとの連続性を設計する

→ 詳細は `docs/20_scene_cards.md` を参照。

---

### 本文生成／執筆
- シーンカードをもとに物語本文を作成する
- 読者体験を最優先し、設計情報は本文に露出させない

※ 完成した本文は GitHub ではなく、ブログ媒体で公開します。

---

### レビュー／評価
- キャラクター整合性
- 感情曲線
- 世界観ルール
- 主題との整合性

といった観点から、生成結果を検証する。

→ レビュー手法の詳細は別途記録します。
