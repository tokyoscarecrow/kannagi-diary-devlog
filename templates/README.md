# Templates（再利用用テンプレート）

このディレクトリには、  
**『かんなびメモランダム』制作で実際に使用・検証された設計手法を、  
第三者が再利用できる形に抽象化したテンプレート**を収録しています。

ここにあるファイルは「完成した設定」ではなく、  
**物語を設計・制御・評価するための“型”**です。

---

## 収録テンプレート一覧

### config_template.yml
プロジェクト全体を制御するための Config テンプレートです。

- 世界観・能力体系・倫理ルール
- 物語構造（大枠）
- 文体・トーンの制約
- 評価軸（レビュー観点）

などを **生成・レビュー・整合性チェックに耐える形**で定義するためのスキーマを提供します。

> 実際の運用例は `raw/config/` を参照してください。

---

### scene_card_template.yml
1シーンを「出来事」ではなく  
**物語上の“機能”として設計するためのシーンカード・テンプレート**です。

このテンプレートでは、以下を重視しています。

- シーンごとの目的（goals / scene_question）
- antagonist を「人物に限定しない」設計
- 感情の遷移（emotional_phase）
- 情報開示の制御（reveal_policy）
- 描く／描かない範囲の明示（scope_control）
- 前後シーンとの連続性（tone_continuity）

AI生成・人力執筆のどちらでも使用でき、  
**「何を書くか」だけでなく「何を書かないか」を制御する**ことを目的としています。

> 実際のシーンカード例は `raw/scene_cards/` を参照してください。

---

## 利用上の注意
- これらのテンプレートは、特定のジャンルや作品に最適化されたものではありません
- 各プロジェクトの規模・目的に応じて、項目の追加・削除・簡略化を行ってください
- テンプレート自体も、運用を通じて更新される前提で設計されています

---

## このディレクトリの位置づけ
- `templates/` ：再利用可能な「型」
- `raw/` ：実際に使われた一次資料（実例）
- `docs/` ：設計思想・判断理由の解説

という役割分担の中で、  
このディレクトリは **再現性の中核**を担います。
